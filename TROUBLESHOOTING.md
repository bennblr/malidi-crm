# Решение проблем

## Проблема: 404 ошибки для статических файлов

**Решение:**
1. Убедитесь, что сервер запущен на правильном порту (проверьте вывод `npm run dev`)
2. Если сервер работает на порту 3001, обновите `NEXTAUTH_URL` в `.env`:
   ```
   NEXTAUTH_URL=http://localhost:3001
   ```
3. Очистите кэш Next.js:
   ```bash
   rm -rf .next
   # или в Windows PowerShell:
   Remove-Item -Recurse -Force .next
   ```
4. Перезапустите сервер

## Проблема: Авторизация не работает

**Решение:**
1. Проверьте, что `NEXTAUTH_SECRET` установлен в `.env`
2. Убедитесь, что `NEXTAUTH_URL` соответствует порту, на котором работает сервер
3. Проверьте логи в консоли браузера и терминале сервера
4. Убедитесь, что пользователь существует в базе данных:
   - Email: `admin@example.com`
   - Пароль: `admin123`

## Проблема: Иконка просмотра пароля не работает

**Решение:**
1. Убедитесь, что используется правильная версия Ant Design
2. Иконка должна быть кликабельной - проверьте в DevTools, что элемент имеет `pointer-events: auto`
3. Если проблема сохраняется, перезапустите сервер разработки

## Общие рекомендации

1. Всегда используйте правильный порт в `NEXTAUTH_URL`
2. После изменения `.env` перезапустите сервер
3. Очищайте кэш `.next` при проблемах со сборкой
4. Проверяйте логи в консоли браузера (F12) и терминале сервера
